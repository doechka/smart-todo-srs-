# Диаграммы поведения системы "Умные карточки"

## Use Case Диаграмма

### Акторы:
- **Пользователь** - основной пользователь системы
- **Система** - автоматические процессы

### Основные Use Cases:

#### Управление контентом:
- Создание колод
- Добавление/редактирование карточек
- Управление тегами

#### Процесс обучения:
- Начало сессии обучения
- Оценка карточек (интервальное повторение)
- Просмотр статистики прогресса

#### Системные функции:
- Валидация данных
- Расчет интервалов повторения
- Автосохранение

## Диаграммы последовательности

### 1. Создание колоды и карточек
Показывает процесс создания новой колоды и добавления первых карточек с валидацией данных.

### 2. Процесс обучения
Демонстрирует полный цикл обучения с алгоритмом интервального повторения (SM2).

### 3. Просмотр статистики
Отображает механизм сбора и отображения статистики обучения.

### 4. Ежедневное обучение
Комплексный сценарий типичного дня пользователя от уведомления до вечерней аналитики.

## Техническая реализация

### Используемые технологии:
- **PlantUML** - для кодового представления диаграмм
- **SQLite** - база данных
- **REST API** - архитектура взаимодействия

### Ключевые компоненты:
- DeckService - управление колодами
- LearningService - логика обучения
- SpacedRepetition - алгоритм интервального повторения
- StatisticsService - аналитика и отчетность
